## 1. 프로젝트 개요

KAIB2026은 대한민국 국가인공지능전략위원회가 공개한 **2026년 AI 예산사업 통합 설명자료(PDF)**에서 데이터를 추출하여 구축한 오픈 분석 플랫폼이다. 총 533개 AI 재정사업, 41개 부처, 확정예산 27조 799억 원 규모의 데이터를 JSON으로 정제하고, 순수 정적 웹(HTML+JS)으로 시각화한다.

**핵심 특징 세 가지**

- **백엔드 없음**: `index.html` 하나가 전부. 별도 서버·DB 불필요.
- **오프라인 동작**: JSON 파일만 있으면 인터넷 없이도 대부분 기능 사용 가능.
- **데이터 분리 설계**: `data/` 폴더의 JSON만 교체하면 다른 연도·다른 정책 데이터에도 그대로 적용 가능.

## 3. 핵심 워크플로우

### 3.1 전체 파이프라인

```
① 원본 PDF
       ↓  (scripts/extract_pdf.py — 저장소에는 미포함)
② budget_raw.json   ← PDF 원시 추출, 533개 사업
       ↓  (scripts/build_db.py — 저장소에는 미포함)
③ budget_db.json    ← 분석 태깅·집계 완료 (핵심 파일)
   similarity_analysis.json
   collaboration_analysis.json
   hybrid_similarity.json
   toc_mapping.json
       ↓  (브라우저 fetch)
④ index.html        ← loadData() → initDashboard()
       ↓
⑤ 10개 탭 렌더링    ← 각 js 모듈 초기화
       ↓
⑥ 내보내기: CSV / XLSX / Markdown / PDF
```

### 3.2 데이터 로딩 흐름 (index.html 내부)

```javascript
// DOMContentLoaded 시 자동 실행
async function loadData() {
    // 1순위: 웹서버 모드 (fetch)
    const resp = await fetch('data/budget_db.json');
    DATA = await resp.json();
    // 실패 시: loadSampleData() → 내장 샘플 데이터로 fallback
    initDashboard();
}

function initDashboard() {
    // 전역 DATA 객체 기반으로 필터·탭 초기화
    // 각 탭 js 모듈은 탭 진입 시 지연 초기화
    initPolicyClusterTab(DATA);   // policy-cluster.js
    initCrossCompareTab(DATA);    // cross-compare.js
    initFutureSimTab(DATA);       // future-sim.js
    initBudgetInsightTab(DATA);   // budget-insight.js
}
```

**중요**: `fetch`는 `file://` 프로토콜에서 CORS 오류로 차단된다. 반드시 로컬 HTTP 서버가 필요하다.

### 3.3 탭별 초기화 시점

| 탭 | 초기화 시점 | 담당 파일 |
|---|---|---|
| 개요 / 부처별 / 분야별 | 앱 시작 시 즉시 | index.html 내부 |
| 유사성 분석 | 탭 첫 진입 시 | index.html 내부 (비동기 fetch) |
| 사업 목록 | 탭 첫 진입 시 | index.html 내부 |
| 정책 클러스터 | `initDashboard()` 시 | policy-cluster.js |
| 비교 분석 | `initDashboard()` 시 | cross-compare.js |
| 미래 시뮬레이터 | `initDashboard()` 시 | future-sim.js |
| 예산 인사이트 | 탭 첫 진입 시 (중복 방지) | budget-insight.js |
| AI 기술 분석 | 탭 진입 시 | ai-tech.js |

## 4. 데이터 포맷 상세

### 4.1 budget_db.json — 핵심 파일

대시보드가 주로 참조하는 파일. 최상위 구조:

```json
{
  "metadata": { ... },    // 전체 통계 요약
  "projects": [ ... ],    // 533개 사업 배열
  "analysis": { ... }     // 9개 분석 섹션
}
```

#### 사업 1건의 주요 필드 (budget_db.json > projects[])

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | int | 순번 1~533 |
| `name` | string | `부처명_사업명` 고유 식별자 |
| `department` | string | 소관 부처 |
| `field` / `sector` | string | 분야 / 부문 |
| `status` | string | `계속` / `신규` |
| `is_rnd` | bool | R&D 사업 여부 |
| `is_informatization` | bool | 정보화 사업 여부 |
| `budget` | object | 연도별 예산 (백만원) |
| `sub_projects` | array | 내역사업 목록 |
| `ai_domains` | string[] | AI 도메인 (27종 자동 분류) |
| `ai_tech` | string[] | AI 기술유형 (16종) |
| `rnd_stage` | string | R&D 단계 (TRL 기반 5종) |
| `keywords` | string[] | 핵심 키워드 |
| `page_start/end` | int | PDF 원본 페이지 |
| `purpose` | string | 사업 목적 (전문) |
| `description` | string | 사업 내용 (전문) |
| `legal_basis` | string | 법적 근거 |

#### budget 객체

```json
"budget": {
  "2024_settlement": 12508.0,   // 2024 결산 (백만원)
  "2025_original":   16111.0,   // 2025 본예산
  "2025_supplementary": 16111.0,// 2025 추경
  "2026_request":    14762.0,   // 2026 부처요구
  "2026_budget":     10300.0,   // 2026 확정 (항상 존재)
  "change_amount":   -5811.0,   // 증감액
  "change_rate":     -36.1      // 증감률 (%)
}
```

### 4.2 budget_raw.json vs budget_db.json 비교

| 항목 | budget_raw.json | budget_db.json |
|---|---|---|
| 크기 | 15MB | 5.4MB |
| 구조 | 단순 배열 | `{metadata, projects, analysis}` |
| 추가 필드 | 없음 | `is_rnd`, `ai_domains`, `ai_tech`, `rnd_stage`, `keywords`, `page_start/end` |
| 분석 데이터 | 없음 | 부처별·분야별 집계, 중복 분석, 키워드 클러스터 |
| 용도 | 추출 원본 보존·디버깅 | 대시보드 메인 데이터 |

### 4.3 유사성 관련 JSON 파일

| 파일 | 크기 | 내용 |
|---|---|---|
| `similarity_analysis.json` | 1.1MB | 인력양성 분야 사업 간 유사도 (Jaccard + TF-IDF + LSA) |
| `collaboration_analysis.json` | 3.2MB | 부처 간 협업 가능성 분석 |
| `hybrid_similarity.json` | 15MB | 전체 사업 하이브리드 유사도 (이전 버전) |
| `asymmetric_pairs_for_collaboration.json` | 479KB | 비대칭 협업 쌍 |
| `toc_mapping.json` | 143KB | PDF 목차→페이지 매핑 (533항목) |

## 5. JS 모듈별 역할

### 5.1 index.html 내부 (인라인)

전체 10,211줄 중 약 8,000줄이 인라인. 핵심 함수 그룹:

| 함수 그룹 | 주요 함수 | 역할 |
|---|---|---|
| 데이터 로딩 | `loadData()`, `initDashboard()` | JSON fetch, 전역 DATA 구성 |
| 공통 유틸 | `formatBillion()`, `getBudget2026()`, `getChangeRate()` | 포맷·계산 헬퍼 |
| 탭 라우팅 | `switchToTab()`, `parseHash()`, `restoreFromHash()` | URL 해시 기반 탭 상태 관리 |
| 개요 탭 | `renderOverview()`, `renderDeptDonut()`, `renderTopChange()` | KPI 카드·차트 렌더 |
| 부처 탭 | `renderDepartment()`, `renderScatterPlot()` | 부처별 산점도·바 차트 |
| 중복 분석 | `analyzeDuplicates()`, `filterHybridPairs()` | Jaccard·TF-IDF 유사도 계산 |
| 사업 목록 | `renderProjects()`, `filterProjects()`, `sortRows()` | 가상 스크롤 테이블 |
| 내보내기 | `downloadMarkdown()`, `exportProjectCSV()`, `exportProjectExcel()` | 결과물 다운로드 |

### 5.2 외부 JS 모듈

#### `policy-cluster.js` (2,606줄) — 정책 테마 클러스터링

10개 정책 테마(디지털전환, 인재양성, AI인프라, 안전·보안 등)를 키워드 가중치로 자동 분류. 각 테마별 예산 현황·사업 목록·부처 분포를 시각화한다.

```javascript
const THEMES = [
  { id: 'digital',  name: '디지털 전환', keywords: [{word:'디지털', weight:3}, ...] },
  { id: 'talent',   name: '인재 양성',   keywords: [{word:'인재', weight:3}, ...] },
  // ... 10개 테마
];
```

#### `future-sim.js` (2,138줄) — 미래 예산 시뮬레이터

2024~2026 실적 기반으로 2027~2030 예산을 4가지 시나리오(현상유지·급성장·선택집중·긴축)로 예측. 부처·도메인·유형별 필터 조합이 가능하다.

#### `cross-compare.js` (890줄) — 부처 간 교차 비교

복수 부처를 선택해 예산 규모·증감률·R&D 비중·AI 도메인 분포를 나란히 비교. 효율성 지표(예산 대비 사업 수)도 제공한다.

#### `budget-insight.js` (994줄) — 예산 인사이트

낭비 위험 스코어링(중복도·증감 이상치·집행률), HHI 예산 집중도 분석, 국회 질의 포인트 자동 생성 3개 서브탭으로 구성된다.

#### `ai-tech.js` (602줄) — AI 기술 분석

16종 AI 기술유형과 5단계 R&D 단계(TRL 기반)별 예산 분포를 막대·도넛 차트로 시각화한다.

#### `network-viz.js` (353줄) — 유사성 네트워크

D3 force-directed graph로 유사 사업 간 연결망을 표현. 노드 크기 = 예산 규모, 엣지 두께 = 유사도.

#### `notes-pdf.js` (391줄) — 메모 관리·PDF 리포트

각 사업 상세 모달에서 메모 작성 가능(localStorage 저장). 필터링된 사업 목록을 PDF 리포트로 생성(html2canvas + jsPDF 방식).

#### `budget-advanced.js` (240줄) — 고급 예산 분석

이상치 탐지(Z-score 기반), HHI 시장집중도 분석, 예산 분포 상세 차트 3가지 위젯을 제공한다.

## 6. 로컬 실행 방법

### 6.1 클론

```bash
git clone https://github.com/hollobit/KAIB2026.git
cd KAIB2026
```

### 6.2 로컬 서버 실행 (필수)

`index.html`을 직접 더블클릭하면 `fetch()` CORS 오류로 JSON을 불러오지 못한다. 반드시 HTTP 서버를 통해 실행해야 한다.

**Python (가장 간단)**

```bash
# Python 3
python3 -m http.server 8080

# 브라우저에서 접속
open http://localhost:8080
```

**Node.js**

```bash
npx serve .
# 또는
npx http-server . -p 8080
```

**VS Code**
Live Server 익스텐션 설치 후 `index.html` 우클릭 → `Open with Live Server`

### 6.3 데이터 확인

서버 실행 후 브라우저 개발자 도구 Console에서:

```javascript
// 로드된 데이터 확인
console.log(DATA.metadata);
console.log(DATA.projects.length);  // 533
```

### 6.4 폴백 동작

JSON fetch가 실패해도 `loadSampleData()`가 내장 샘플 데이터(약 10개 사업)를 주입해 UI를 최소 동작시킨다. 전체 기능은 JSON 파일이 있어야 한다.

## 8. HWPX 연계 확장 방향

*이 섹션은 향후 작업 로드맵이다.*

### 8.1 현재 상황

원본 데이터 소스는 PDF 문서이나, 동일 양식의 **HWPX 파일**이 존재한다. 이를 활용하면 PDF 추출 오류 없이 더 정확한 데이터를 얻을 수 있다.

### 8.2 검토 방향: 양방향 파이프라인

```
방향 A: HWPX → JSON (데이터 수집)
  HWPX 양식 분석 → 섹션·필드 감지 → JSON 스키마 매핑 → budget_raw.json 생성

방향 B: JSON → HWPX (문서 생성)
  budget_db.json → HWPX 템플릿 채우기 → 사업별 표준 보고서 자동 생성
```

### 8.3 구체 작업 항목

**방향 A (HWPX → JSON)**

- HWPX 양식의 슬롯 구조 분석 (□ ○ ― 기호 기반)
- 사업코드, 부처, 예산 필드 자동 감지 규칙 작성
- `budget_raw.json` 스키마에 맞는 변환 스크립트 작성

**방향 B (JSON → HWPX)**

- `budget_db.json`의 1개 사업 → HWPX 1페이지 매핑 설계
- HWPX 스킬의 `config.yaml` 슬롯 정의
- 533개 사업 일괄 생성 파이프라인 구성

### 8.4 우선순위 권장

데이터 정확도와 재현 가능성을 위해 **방향 A를 먼저** 구현한 후, 방향 B로 리포트 자동화를 구현하는 순서가 효율적이다.
