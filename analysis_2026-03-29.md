# 7. hybrid_similarity.json_backup 역분석 및 로직 재설정

## 1) hybrid_similarity.json_backup 파일 상태
- 실제 파일이 비어 있음(내용 없음)
- 구조/예시 데이터가 없어 기존 hybrid 분석 결과의 구체적 기준을 추정할 수 없음

## 2) hybrid 분석 로직 재설정 가이드
- 기존 similarity/collaboration 분석은 텍스트 유사도, 부서, 도메인, 키워드 등을 기준으로 페어/클러스터를 생성함
- hybrid 분석은 "융합형/복합형/다중 도메인" 사업 간의 유사성 또는 연계성을 별도로 추출하는 것이 목적일 가능성이 높음
- 예시 기준(가설):
    - ai_domains가 2개 이상이거나, keywords에 '융합', '복합', '하이브리드' 등 포함
    - 또는 field/sector/keywords/overview 등에서 융합 관련 용어가 다수 등장
- 분석 로직 예시:
    1. projects에서 ai_domains가 2개 이상인 사업만 추출
    2. 이들 간의 유사도(텍스트, 키워드, 예산 등)로 페어/클러스터링
    3. 또는 keywords에 '융합', '복합', 'hybrid' 등 포함된 사업만 별도 추출
    4. 결과를 hybrid_similarity.json에 저장

## 3) ai_domains 값의 결정 방식
- budget_db.json 내 ai_domains는 각 사업별로 리스트 형태로 입력되어 있음
- 코드/스크립트 내에서 ai_domains를 자동 분류/추출하는 로직은 없음(직접 매핑/수동입력 또는 원본 데이터에서 매핑 추정)
- 즉, ai_domains는 엑셀/원본 데이터 입력 시 담당자가 직접 지정하거나, 외부 매핑표/룰셋에 의해 부여된 것으로 보임
- generate_ai_analysis.py 등 분석 스크립트에서는 ai_domains를 그대로 참조만 하며, 생성/분류하지 않음

## 4) 향후 구현 방향
- hybrid 분석을 위해서는 ai_domains, keywords, overview 등에서 융합/복합/다중 도메인 사업을 정의하는 기준을 명확히 해야 함
- 예시 기준을 코드로 구현하여 hybrid_similarity.json을 자동 생성하도록 개선 필요
- 실제 hybrid 분석 결과 예시가 확보되면, 구조/필드/페어링 방식에 맞춰 추가 개선 가능

# hybrid_similarity.json 미생성 원인 및 생성 방안 분석 (2026-03-29)

## 1. 현황 요약
- `generate_ai_analysis.py`는 budget_db.json을 분석하여 similarity_analysis.json, collaboration_analysis.json을 생성함.
- `rebuild_embedded.py`는 web/data/hybrid_similarity.json을 web/js/embedded-hybrid-data.js로 변환하는 매핑이 있으나, 실제 hybrid_similarity.json이 data 폴더에 존재하지 않음.
- hybrid_similarity.json_backup은 있으나, 내용이 비어 있음.

## 2. 기존 analysis 생성 로직 비교
- similarity_analysis.json, collaboration_analysis.json은 generate_ai_analysis.py에서 직접 생성됨.
- hybrid_similarity.json은 해당 스크립트에서 생성 로직이 없음(코드 내 생성/저장 부분이 존재하지 않음).
- rebuild_embedded.py는 hybrid_similarity.json이 존재할 경우에만 js로 변환함(없으면 skip).

## 3. hybrid_similarity.json의 역할 및 필요성
- (추정) hybrid_similarity.json은 기존 유사/협업 분석과는 다른 기준(예: 하이브리드/융합형 사업, 다중 도메인, 복합 키워드 등)에 대한 유사성 분석 결과를 담는 파일로 설계된 듯함.
- 하지만 generate_ai_analysis.py에는 하이브리드 분석/클러스터링/추출 로직이 구현되어 있지 않음.

## 4. 미생성 원인
- generate_ai_analysis.py에 hybrid(융합/복합) 유사성 분석 및 hybrid_similarity.json 저장 코드가 없음.
- hybrid_similarity.json_backup은 수동/임시로 만들어진 파일로, 자동 생성과 무관.
- rebuild_embedded.py는 파일이 없으면 skip만 하고, 생성은 하지 않음.
